# Experiment 1 – NULL Pointer Dereference (APT Python Bindings)

## Patch Source

Patch URL:
https://launchpadlibrarian.net/764050984/0001-Fix-invalid-nullptr-dereference-in-TagSection.keys.patch

Affected component: `python/tag.cc` (APT Python bindings)

Patch type detected: `null_deref`

---

## Vulnerability

The function `PyString_FromStringAndSize()` may return `nullptr` when
invalid Unicode input is encountered. The original code used the returned
object without a NULL check, causing a crash. The fix introduces an explicit
check and returns safely.

---

## Extracted Code Fragments

```cpp
// Vulnerable lines (−)
PyObject *Obj = PyString_FromStringAndSize(Start, End-Start);
PyList_Append(List, Obj);

// Fix lines (+)
if (Obj == nullptr)
    return Py_DECREF(List), nullptr;
```

## How the True Positive Was Found

The libwebp match passed verification because:

1. **Language cross-check**: `.c` file, patch is `null_deref` (C/C++) ✓
2. **Vulnerable pattern present**: `PyString_FromStringAndSize(` in context ✓
3. **Fix pattern absent**: no `if (Obj == nullptr)` in the same 5-line window ✓

The vim and SWIG binding matches failed step 3 — both already perform
null checks in their local context, so they are correctly excluded.

---

## Key Finding

**Context pairs are the highest-value signature type.** The two-line join
`PyString_FromStringAndSize( | if (Obj == nullptr)` almost never appears
coincidentally in unrelated code. In this experiment, all confirmed candidates
were found via context pair signatures. Individual API calls (`PyString_FromStringAndSize(`)
returned 4 candidates with high noise; the context pair narrowed those 4
to 1 confirmed true positive.

Searching the API call alone produced F1 ≈ 0.40. Adding the context pair
verification raised precision to 1.0 on this candidate set.
